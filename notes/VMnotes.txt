-Access the bash config files to alter logging setting
-nano ~/.bashrc
	- Ensure that the following are true:
		-(These can be increased they just determine character limits) 
		  HISTSIZE=5000
		  HISTFILESIZE=10000
		-shopt -s histappend is present, this means that the history is appended instead of replaced.
		- add these two lines to allow for logging of all users and instances correctly.
		  export PROMPT_COMMAND='RETRN_VAL=$?;logger -p local6.debug "$(whoami) [$$]: $(history 1 | sed "s/^[ ]*[0-9]\+[ ]*//" ) [$RETRN_VAL]"'
		  export PROMPT_COMMAND="history -a; history -c; history -r; $PROMPT_COMMAND"
		- save the file as is and run the following command
			source ~/.bashrc
		- Now we neeed to access the following
			  sudo -e /etc/rsyslog.d/bash.conf
		-its will be blank add this line:
				local6.*    /var/log/commands.log
		-Run this command to reset the service to log it
			  sudo service rsyslog restart 
-To test this feature, you can log out then back in, run the history command and you should see your bash commands
